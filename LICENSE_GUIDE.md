# License Guide

Plain-English guidance for choosing a license.

## CC0
**Best for:** maximum freedom and zero friction.

**Buyers can:**
- Use, modify, and sell the asset in any project.
- Skip attribution.

**Buyers canâ€™t:**
- Claim they made your original asset.

**Common mistakes:**
- Using CC0 when you still want credit.

## CC-BY
**Best for:** wide usage with required credit.

**Buyers can:**
- Use, modify, and sell the asset.
- Include it in commercial work.

**Buyers must:**
- Give attribution in a reasonable place (README, credits, or product page).

**Common mistakes:**
- Not stating where attribution should go.

## MIT
**Best for:** code or code-heavy assets.

**Buyers can:**
- Use, modify, and sell the code.
- Include it in commercial products.

**Buyers must:**
- Keep the MIT license text with the code.

**Common mistakes:**
- Using MIT for non-code assets without explaining how it applies.

## Quick Pick
- **Want zero friction?** Use **CC0**.
- **Want credit?** Use **CC-BY**.
- **Shipping code?** Use **MIT**.
